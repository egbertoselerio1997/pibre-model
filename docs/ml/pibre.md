# PIBRe Model Summary

## 1. Title and model summary

The pibre module implements a Physics-Informed Bilinear Regression model in the measured-output space generated by the ASM1 simulation workflow. The model learns an unconstrained bilinear mapping from operating conditions plus influent measured composites to effluent measured composites, then projects those raw predictions onto the invariant subspace defined by the measured-space constraint matrix.

In this repository the measured outputs are COD, TSS, TN, TP, NH4-N, NO3-N, PO4-P, and alkalinity. The learned model therefore predicts only quantities that are directly compatible with the constraint matrix computed in the notebook.

## 2. Background and use case

Activated-sludge simulations provide mechanistically consistent datasets, but a direct mechanistic solver is still more expensive than a compact surrogate model. A purely statistical regressor can approximate the simulated input-output map, yet it may violate conservation relations inherited from the stoichiometric structure of the process.

PIBRe addresses that gap by separating the learning problem into two parts:

1. a flexible bilinear regression layer that approximates the nonlinear process response
2. an orthogonal projection layer that enforces the measured-space invariants defined by the ASM1 stoichiometric mapping

This makes the model useful as a surrogate when fast prediction is required but the results must remain consistent with the conservation relations represented by the measured-space A matrix.

## 3. Mathematical definition

Let $x \in \mathbb{R}^{M}$ be the feature vector containing hydraulic and aeration variables together with measured influent composites, and let $C_{in} \in \mathbb{R}^{K}$ be the influent measured composite vector. In this repository $M = 10$ and $K = 8$.

The unconstrained bilinear predictor is

$$
C_{raw} = W_L x + b + x^T W_B x
$$

where $W_L \in \mathbb{R}^{K \times M}$ is the linear weight matrix, $b \in \mathbb{R}^{K}$ is the bias vector, and $W_B \in \mathbb{R}^{K \times M \times M}$ is the bilinear tensor.

The measured-space invariants are encoded by a constraint matrix $A \in \mathbb{R}^{C \times K}$ derived in the notebook from the null space of the macroscopic stoichiometric matrix. The physically compliant prediction is obtained by the orthogonal projection

$$
C^{*} = C_{raw} - A^T (A A^T)^{-1} A (C_{raw} - C_{in})
$$

This guarantees

$$
A C^{*} = A C_{in}
$$

up to numerical precision.

- Out_TP
- Out_NH4_N
- Out_NO3_N
- Out_PO4_P
- Out_Alkalinity

Key assumptions:

- the A matrix is computed in measured-output space and therefore the machine-learning target space must exactly match the measured outputs
- influent measured composites are derived from the simulated influent state vector using the explicit ASM1 composition matrix rather than being taken from separate laboratory data
- feature scaling is applied to the inputs, while the predicted outputs remain in physical units so the projection layer acts directly in the measured concentration space

## 5. Implementation used in this repository

Implementation is in src/models/ml/pibre.py.

The exact repository workflow is:

1. build measured-space supervised features from the ASM1 dataset and composition matrix
2. split the dataset into train, validation, and test partitions using reusable preprocessing helpers
3. scale the feature matrix with a standard scaler fitted on the training split
4. run Optuna hyperparameter tuning using reusable utility helpers
5. train the final bilinear model with the selected hyperparameters
6. evaluate both raw and projected predictions using shared metric utilities
7. optionally save a pickle model bundle plus JSON metrics and Optuna summaries under the configured results paths

The saved model bundle contains the learned state dictionary, feature and target column order, scaling objects, A matrix, and the selected hyperparameters.

## 6. Architecture details and adopted standard architecture name

The adopted architecture is a bilinear regression model with a differentiable projection layer. It is not a deep multilayer neural network. Instead, it is a compact second-order model designed to capture multiplicative interactions without introducing a hidden-state stack.

The architecture has two explicit components:

1. a linear layer that captures first-order effects
2. a bilinear tensor contraction that captures pairwise multiplicative interactions among all inputs

The projection layer is not learned. It is computed analytically from the measured-space constraint matrix and registered as a fixed operator inside the PyTorch module.

## 7. Training or optimization notes

Hyperparameter tuning is performed with Optuna and uses reusable utilities rather than ad hoc notebook code. The tuned quantities are:

- learning rate
- L1 regularization strength
- optimizer weight decay
- gradient clipping norm
- bilinear-weight initialization scale

Training uses the Adam optimizer and minimizes the projected prediction mean squared error plus an L1 penalty on the learned linear and bilinear weights. The repository parameter file defines multiple tuning profiles so notebook usage can choose fast, balanced, or thorough optimization without changing source code.

## 8. Prediction workflow

Prediction in this repository follows the contract below:

1. prepare the test dataset in measured-output space
2. load the persisted pickle bundle
3. reconstruct the trained PIBRe module and feature scaler
4. transform the feature columns with the saved scaler
5. compute the raw bilinear prediction and the projected physically compliant prediction
6. return predictions aligned to the supplied test rows

The notebook pipeline also reports aggregate metrics, per-target metrics, and summary statistics of the mass-balance residuals.

## 9. Limitations and expected failure modes

Important limitations:

- the model is only as informative as the simulated dataset used for training and does not replace mechanistic calibration against plant data
- projection guarantees consistency with the chosen measured-space invariants, but it does not guarantee correctness for phenomena not represented in the measured outputs
- the bilinear form is expressive but still low-order; if future tasks require stronger nonlinearity, a richer surrogate architecture may be necessary
- prediction quality can degrade if the model is extrapolated far outside the operating and influent ranges used by the ASM1 simulation workflow

Expected failure modes:

- poor predictive accuracy if the dataset is too small or the Optuna budget is too low
- unstable training if hyperparameters are pushed far outside the configured search space
- misleading use if the A matrix and target columns are not kept in the same measured-output basis
